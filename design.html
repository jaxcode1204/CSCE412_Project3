Text Description

The request handler creates requests with input IP, output IP, task time, and job type. Each request is sent to the load balancer. The load balancer first checks the firewall, then places allowed requests into a queue. Idle web servers pull the next request from the queue, process it for the required number of clock cycles, and then ask for another task.

The load balancer keeps track of time and adjusts the number of servers to avoid overload or underuse. If the queue falls below 50 times the number of servers, one server is removed. If the queue grows above 80 times the number of servers, one server is added. After each scaling change, the program waits the configured number of rest cycles before checking again.

Classes and Data

  • Request: struct holding IP in, IP out, time, job type, and task number.
  • Firewall: blocks configured IP ranges.
  • WebServer: processes one request at a time.
  • Server: container for all web servers.
  • LoadBalancer: manages queue, time, scaling, and logging.
